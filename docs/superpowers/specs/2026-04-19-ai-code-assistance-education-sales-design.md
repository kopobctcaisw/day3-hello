# AI 코드어시스턴스 교육 프로그램 판매 랜딩페이지 설계서

- 작성일: 2026-04-19
- 프로젝트 유형: 단일 정적 랜딩페이지
- 목적: 카카오톡 무료 상담 신청 유도 후 결제 전환
- 타깃: 40대 이상 성인, 비전공 포함
- 핵심 메시지: **비전공 40대도 AI 코딩으로 실무 전환 가능**
- 핵심 CTA: **카톡으로 무료 상담 받기**

## 1) 목표와 성공 기준

### 목표
- 방문자가 프로그램 가치를 빠르게 이해하고 신뢰를 형성한다.
- 랜딩페이지 내 CTA 클릭을 통해 카카오 오픈채팅 상담으로 유입한다.

### 성공 기준(초기)
- 페이지 내 CTA 클릭률(상/중/하단 합산) 측정 가능 구조 확보
- 핵심 정보(성과, 프로그램, 방식/기간/가격)를 스크롤 1회 내 식별 가능

## 2) 정보 구조(성과 증명형)

1. 히어로 섹션
   - 헤드라인: 비전공 40대도 AI 코딩으로 실무 전환 가능
   - 서브카피: 나이·전공 제약 없이 실무 중심으로 학습 가능한 점 강조
   - CTA 버튼: 카톡으로 무료 상담 받기

2. 수강생 성과 사례 섹션(가장 먼저 배치)
   - 전/후 변화 카드(예: 업무 시간 단축, 산출물 완성, 프로젝트 수행)
   - 수치와 기간 중심 표현

3. 프로그램 소개 섹션
   - 무엇을 배우는지(핵심 역량)
   - 어떤 문제를 해결하는지(실무 적용 관점)

4. 수강 정보 섹션
   - 수강 방식
   - 기간
   - 가격(즉시 공개)
   - 포함 항목 및 안내 문구

5. 하단 CTA 섹션
   - CTA 재강조 및 카카오 오픈채팅 링크 반복 노출

## 3) 콘텐츠/카피 원칙

- 톤앤매너: 신뢰 + 동기부여 균형
- 과장/추상 표현보다 구체적 사실 우선
- 성과는 기간·수치·산출물 중심으로 제시
- 가격은 금액만이 아니라 포함 범위를 함께 제시
- CTA 문구는 전 구간 동일하게 유지

## 4) 기술 아키텍처

- 파일 구성: `index.html` 단일 파일(필요 시 같은 파일 내 `<style>`, `<script>`)
- 기술 스택: HTML/CSS/Vanilla JS
- 배포 방식: 정적 호스팅 전제
- 의존성: 외부 빌드 도구 및 프레임워크 미사용

## 5) 컴포넌트 경계

- Hero
- ProofCards(성과 카드 목록)
- ProgramOverview
- PricingInfo
- CTARepeater(상/중/하 CTA)

각 섹션은 독립적으로 수정 가능하게 마크업 구획을 분리한다.

## 6) 데이터 흐름

- 정적 콘텐츠 렌더링(서버 데이터 호출 없음)
- 사용자 행동 흐름:
  1) 랜딩 유입
  2) 성과 사례 확인
  3) 프로그램/가격 확인
  4) CTA 클릭
  5) 카카오 오픈채팅 이동

- 이벤트 추적(선택): CTA 위치별 클릭 이벤트 구분 가능하도록 `data-cta-position` 속성 부여

## 7) 오류/예외 처리

- 카카오 링크 누락 방지: 모든 CTA 앵커의 `href` 동일값 검증
- 링크 열기 실패 대비: 버튼 인접 위치에 링크 텍스트 보조 노출 가능
- 콘텐츠 예외: 성과 데이터가 부족한 경우, 정량 지표 대신 구체 사례 설명으로 대체

## 8) 접근성/가독성

- 모바일 우선 반응형
- 40대 이상 가독성을 위해 본문/헤드라인 크기 여유 있게 설정
- 대비 충분한 색상 조합, 터치 영역 44px 이상 권장
- 핵심 CTA는 스크롤 중 반복 노출

## 9) 테스트 범위

- 레이아웃 테스트: 모바일(360px), 태블릿, 데스크톱
- 링크 테스트: 상/중/하 CTA 모두 동일 카카오 링크로 이동
- 콘텐츠 테스트: 핵심 메시지, 성과, 가격 정보 노출 확인
- 기본 품질 테스트: HTML 유효성, 콘솔 에러 없음

## 10) 범위 제외(YAGNI)

- 다중 페이지 확장
- 로그인/회원 기능
- 결제 시스템 직접 연동
- 백오피스/CRM 자동 연동

초기 버전은 상담 전환에 필요한 핵심 정보 전달과 CTA 최적화에 집중한다.
